# Estructura del Portfolio

Este portfolio está organizado por **áreas, subáreas y especializaciones** dentro del campo de la informática y la ciberseguridad.

Cada área contiene conocimiento teórico, procesos técnicos y laboratorios prácticos para comprender cómo funcionan las tecnologías y cómo pueden ser utilizadas o analizadas desde una perspectiva de seguridad.

## Significado de cada sección

Cada especialización contiene cuatro carpetas principales que siguen una metodología de aprendizaje estructurada.

### Temas
Contiene el **tema o concepto técnico que se está estudiando**.

Aquí se documentan conceptos fundamentales como protocolos, técnicas, herramientas o tecnologías.

Ejemplos:
- ARP
- DNS
- Network Scanning
- Memory Forensics
- Firewall

---

### Explicación
Explica **cómo funciona el concepto desde un punto de vista técnico**.

Esta sección incluye:
- teoría
- funcionamiento interno
- contexto técnico
- implicaciones de seguridad

El objetivo es comprender **por qué y cómo funciona la tecnología**.

---

### Procesos
Describe **el proceso técnico paso a paso**.

Incluye:

- configuraciones
- comandos utilizados
- flujo del sistema
- análisis técnico

Esta sección explica **cómo se implementa o analiza el concepto en la práctica**.

---

### Laboratorios
Contiene **laboratorios prácticos y simulaciones**.

Aquí se aplican los conceptos aprendidos mediante:

- entornos virtuales
- simulaciones de red
- análisis de tráfico
- pruebas de seguridad
- escenarios de ataque y defensa

Los laboratorios pueden incluir herramientas como:

- Kali Linux
- Wireshark
- Nmap
- GNS3
- Cisco Packet Tracer
- máquinas virtuales

---

## Objetivo del portfolio

El objetivo de este portfolio es desarrollar una comprensión completa de las tecnologías estudiadas mediante tres niveles:

1. Comprender el concepto.
2. Analizar cómo funciona.
3. Aplicarlo en un laboratorio práctico.

Este enfoque permite estudiar cada tema desde una perspectiva **teórica, técnica y práctica**.
